# FocusFlow - Project Progress (as of 2026-05-21)

## Current Version: v0.2.0 (Phase 1 complete, Phase 2 partially integrated)

### What Works (Stable)
- Core timer engine (Protocol A & B) with pause/resume (Protocol B only), auto‑trough after 3 min pause, micro‑pauses (10 sec blinks with 3 sec pre‑alert and end ding).
- Ghost Mode bar: phase colors, progress, abort button always visible, skip only on micro_rest/trough, pause only on Protocol B deep block.
- Persistence: localStorage backup works; session restores correctly on reload (no flash).
- Authentication: email/password + Google OAuth via Supabase, user profile creation (age, chronotype, life expectancy) works.
- Analytics dashboard shows “No Data Yet” because sessions are not yet saved to Supabase (profile exists but sessions missing).

### What Is Broken / Not Yet Working
- **Sessions not saving to Supabase** – The Supabase `sessions` and `phases` tables exist but inserts fail with:
  - `Could not find the 'updated_at' column of 'sessions' in the schema cache` (column missing from table)
  - `there is no unique or exclusion constraint matching the ON CONFLICT specification` (phases table lacks unique constraint on (session_id, phase_index))
- **Web Push Notifications** – Not yet implemented (VAPID keys not generated, Edge Functions not deployed).
- **Real‑time sync** – Not yet implemented (supabase subscriptions not used).
- **Desktop (Tauri)** – Not yet started.

### Immediate Next Steps (to fix Supabase saving)
1. Add `updated_at` column to `sessions` table (run ALTER TABLE).
2. Add unique constraint on `phases` table for `(session_id, phase_index)`.
3. Re‑run the migration or apply fixes manually in Supabase SQL editor.
4. Test session creation again; verify rows appear in Supabase.
